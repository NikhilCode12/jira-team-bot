# Jira Feedback to Issue

Create Jira bugs from product feedback using a free LLM (Groq) and Jira Cloud API.

## Setup

1. Copy `.env.example` to `.env` and fill in:
   - **JIRA_BASE_URL**, **JIRA_EMAIL**, **JIRA_API_TOKEN** (from [Atlassian API tokens](https://id.atlassian.com/manage-profile/security/api-tokens))
   - **GROQ_API_KEY** (from [Groq Console](https://console.groq.com))

2. Optional: Adjust hardcoded defaults in `.env` (Epic, Labels, Module, Environment, Epic field ID).

3. Install and run:

   ```bash
   cd jira-feedback-to-issue
   pip install -r requirements.txt
   uvicorn app:app --reload
   ```

4. Open http://127.0.0.1:8000

## Jira project setup

- **Project**: ZRA (hardcoded).
- **Components**: Loaded from Jira via API; choose in the dropdown.
- **Epic**: ZRA-51 (hardcoded). If your project does not use Epic Link, set `JIRA_EPIC_FIELD_ID=` in `.env` to skip.
- **Issue type**: Bug.
- **Labels**: BetaConnect.
- **Module / Environment**: Set in description (Module: Super Admin, Environment: Production). Custom fields can be added later via env if your project has matching field IDs.

## Usage

1. Enter feedback in the text area (required).
2. Optionally set Sprint, Component, Priority, Customer Reported Bug, Customer Name.
3. Click **Create Jira**. Summary and description are generated by the LLM, then the issue is created.
4. Use **Open in new tab** to go to the new issue.
